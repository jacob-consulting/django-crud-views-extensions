# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions CI (tests + lint) and a tag-triggered PyPI publish pipeline to `django-crud-views-extensions`, modeled on the sibling `django-crud-views` repo's pipeline.

**Architecture:** Three workflow files under `.github/workflows/` (`tests.yml`, `lint.yml`, `publish.yml`), all uv-native (`astral-sh/setup-uv@v8`, no separate `actions/setup-python`). `pyproject.toml` gains `ruff`, `pytest-cov`, `bump-my-version` as dev dependencies plus `[tool.coverage.run]` and `[tool.bumpversion]` config sections. The one non-obvious wrinkle every CI-facing command must respect: `[tool.uv.sources]` points `django-crud-views` at a local sibling checkout (`../django-crud-views`) that does not exist on a GitHub Actions runner — every `uv run`/`uv sync` invoked in CI (or in verification steps below) must pass `--no-sources` so it resolves `django-crud-views` from PyPI instead.

**Tech Stack:** GitHub Actions, `uv` (dependency management, build), `ruff` (lint/format), `pytest` + `pytest-cov` (tests/coverage), Codecov, `bump-my-version` (release versioning), `pypa/gh-action-pypi-publish` (trusted publishing).

## Global Constraints

- Every CI step that runs `uv run` or `uv sync` against this project MUST include `--no-sources`, because `[tool.uv.sources]` in `pyproject.toml` overrides `django-crud-views` to a local path (`../django-crud-views`) that doesn't exist outside this developer's machine. Omitting `--no-sources` will make the step fail on GitHub's runner.
- After any local verification command that used `--no-sources` (which rewrites `uv.lock` to the PyPI resolution), always finish the task with a plain `uv sync --extra dev` (no `--no-sources`) before committing, so the committed `uv.lock` reflects local-dev (editable-source) state, matching this repo's existing convention.
- Workflow YAML action versions to use: `actions/checkout@v5`, `astral-sh/setup-uv@v8`, `codecov/codecov-action@v5`, `pypa/gh-action-pypi-publish@release/v1`.
- Python version matrix for tests/publish: `["3.12", "3.13", "3.14"]` (matches `requires-python = ">=3.12"` in `pyproject.toml`).
- No `docs.yml` workflow, no Django version matrix, no `fail_under` coverage gate — explicitly out of scope per the design spec (`docs/superpowers/specs/2026-07-14-ci-cd-pipeline-design.md`).

## Task 1: `pyproject.toml` — dev tooling and config

**Files:**
- Modify: `pyproject.toml`
- Modify: `uv.lock` (regenerated by `uv sync`, not hand-edited)

**Interfaces:**
- Produces: dev-installable `ruff`, `pytest-cov`, `bump-my-version`; `[tool.coverage.run]` with `source = ["crud_views_widget_datetimepicker"]`; `[tool.bumpversion]` with `current_version = "0.1.0"`, `tag_name = "v{new_version}"`. Tasks 2-4 assume these are present (Task 2 assumes `pytest-cov` and the coverage source config exist; Task 3 assumes `ruff` is installed).

- [ ] **Step 1: Add `ruff`, `pytest-cov`, `bump-my-version` to the `dev` extra**

Edit `pyproject.toml`:

```toml
[project.optional-dependencies]
dev = [
    "pytest",
    "pytest-django",
    "pytest-mock",
    "django-pipeline",
    "pytest-cov",
    "ruff",
    "bump-my-version",
]
```

(`ruff` was previously undeclared here even though `CLAUDE.md` documents `uv run ruff check .` as the standard lint command — this closes that gap.)

- [ ] **Step 2: Add coverage and bumpversion config**

Append to `pyproject.toml` (after the existing `[tool.pytest.ini_options]` block):

```toml
[tool.coverage.run]
source = ["crud_views_widget_datetimepicker"]

[tool.bumpversion]
current_version = "0.1.0"
commit = true
tag = true
tag_name = "v{new_version}"

[[tool.bumpversion.files]]
filename = "pyproject.toml"
search = 'version = "{current_version}"'
replace = 'version = "{new_version}"'
```

- [ ] **Step 3: Sync the environment**

Run: `uv sync --extra dev`
Expected: resolves and installs `ruff`, `pytest-cov`, `bump-my-version` (and their transitive deps) in addition to the existing dev deps; `django-crud-views` stays as the local editable source (no `--no-sources` here, so `uv.lock` ends up in the normal local-dev state).

- [ ] **Step 4: Verify ruff works**

Run: `uv run --no-sources ruff format --check .`
Expected: exits 0, no output (codebase is already formatted).

Run: `uv run --no-sources ruff check .`
Expected: `All checks passed!`

- [ ] **Step 5: Verify pytest-cov works with the coverage config**

Run: `uv run --no-sources --extra dev pytest --cov=crud_views_widget_datetimepicker --cov-report=term`
Expected: `28 passed`, followed by a coverage table restricted to `src/crud_views_widget_datetimepicker/*` files only (no `tests/` or dependency files listed).

- [ ] **Step 6: Verify bump-my-version config (dry run only)**

Run: `uv run --no-sources bump-my-version bump --dry-run --verbose --allow-dirty patch`
Expected: output shows it would change `tool.bumpversion.current_version` and `project.version` from `0.1.0` to `0.1.1`, and would commit with message `Bump version: 0.1.0 → 0.1.1` and tag `v0.1.1`. (`--allow-dirty` is needed only because the working tree has uncommitted changes at this point in the task — do not use `--allow-dirty` for a real release later.) Confirm no files were actually modified: `git status --short` still shows only `pyproject.toml` and `uv.lock` changed (nothing from bump-my-version itself, since this was `--dry-run`).

- [ ] **Step 7: Restore `uv.lock` to normal local-dev state and clean up artifacts**

Run: `uv sync --extra dev` (plain, no `--no-sources` — undoes the source-override churn from steps 4-6)
Run: `rm -f coverage.xml .coverage`
Run: `git status --short`
Expected: only `pyproject.toml` and `uv.lock` show as modified; no other untracked files.

## Task 2: `.github/workflows/tests.yml`

**Files:**
- Create: `.github/workflows/tests.yml`

**Interfaces:**
- Consumes: `pytest-cov` and `[tool.coverage.run]` from Task 1.

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/tests.yml`:

```yaml
name: Tests

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  tests:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.12", "3.13", "3.14"]
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v8
        with:
          python-version: ${{ matrix.python-version }}
          enable-cache: true
      - run: uv run --no-sources --extra dev pytest --cov=crud_views_widget_datetimepicker --cov-report=xml
      - uses: codecov/codecov-action@v5
        if: matrix.python-version == '3.13'
        with:
          token: ${{ secrets.CODECOV_TOKEN }}
          files: coverage.xml
```

(`--extra dev` is required here: `ruff` is a dev-only dependency, and without it `uv run` only
syncs the base dependency group, so `ruff` is never installed and the step fails with
"Failed to spawn: `ruff`" on a clean runner. Verified during Task 3 review.)

## Task 4: `.github/workflows/publish.yml`

**Files:**
- Create: `.github/workflows/publish.yml`

**Interfaces:**
- Consumes: the same test/lint commands verified in Tasks 2-3.

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/publish.yml`:

```yaml
name: Publish

on:
  push:
    tags:
      - "v*"

jobs:
  tests:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.12", "3.13", "3.14"]
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v8
        with:
          python-version: ${{ matrix.python-version }}
          enable-cache: true
      - run: uv run --no-sources --extra dev pytest --cov=crud_views_widget_datetimepicker --cov-report=xml
      - uses: codecov/codecov-action@v5
        if: matrix.python-version == '3.13'
        with:
          token: ${{ secrets.CODECOV_TOKEN }}
          files: coverage.xml

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v8
        with:
          python-version: "3.13"
          enable-cache: true
      - run: uv run --no-sources --extra dev ruff format --check .
      - run: uv run --no-sources --extra dev ruff check .

  publish:
    needs: [tests, lint]
    runs-on: ubuntu-latest
    environment: pypi
    permissions:
      id-token: write
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v8
        with:
          python-version: "3.13"
      - run: uv build
      - uses: pypa/gh-action-pypi-publish@release/v1
```

- [ ] **Step 2: Validate YAML syntax**

Run: `uv run --with pyyaml python3 -c "import yaml; yaml.safe_load(open('.github/workflows/publish.yml'))"`
Expected: no output, exit 0.

- [ ] **Step 3: Verify the build step locally**

Run: `uv build`
Expected: exits 0; creates `dist/django_crud_views_extensions-0.1.0.tar.gz` and `dist/django_crud_views_extensions-0.1.0-py3-none-any.whl` (exact filenames depend on the current `version` in `pyproject.toml`).

Run: `ls dist/`
Expected: both files listed above are present.

Run: `rm -rf dist/`
Expected: cleans up the local build artifacts (not committed to git).

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/publish.yml
git commit -m "ci: add publish workflow"
```
